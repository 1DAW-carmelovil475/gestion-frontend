1 Introducción y Visión General

1.1 Propósito del Sistema

Hola Informática es una plataforma web integral de gestión empresarial diseñada específicamente para empresas de servicios informáticos. El sistema permite gestionar de forma centralizada todos los aspectos operativos del negocio: empresas clientes, inventario de dispositivos, tickets de soporte técnico, registro de horas de trabajo, comunicación interna mediante chat y análisis estadístico del rendimiento del equipo.

Objetivo principal

Centralizar la gestión operativa de una empresa de servicios IT en una única plataforma accesible desde navegador web, ofreciendo interfaces diferenciadas para el equipo interno y los clientes externos.

2.2 Frontend

  • Tipo: Single Page Application (SPA)
  • Framework: React 18.2 con Vite 5.0 como bundler
  • Enrutamiento: React Router DOM v6 con rutas protegidas por rol
  • Gestión de estado: React Context API con tres contextos: AuthContext, ThemeContext y ChatNotificationsContext
  • Comunicación con backend: Fetch API nativa con wrapper personalizado (apiFetch) que inyecta tokens JWT y gestióna su renovación automática

2.3 Backend

  • Framework: Express.js 4.21 sobre Node.js
  • Arquitectura: API REST organizada en rutas modulares con middleware de autenticación
  • Subida de archivos: Multer para gestión de multipart/form-data
  • Email: Nodemailer para notificaciones automáticas por correo
  • Exportación: XLSX para generación de archivos Excel

2.4 Base de Datos y Servicios Cloud

  • Base de datos: PostgreSQL gestiónada por Supabase
  • Autenticación: Supabase Auth (generación y validación de JWT)
  • Almacenamiento: Supabase Storage (bucket ticket-archivos)
  • SDK: Supabase JS v2.49 en el backend

2.5 Aplicación Móvil

Existe adicionalmente un proyecto de aplicación móvil (gestión-frontend-movil) desarrollado con React Native, que ofrece acceso a las funcionalidades principales desde dispositivos iOS y Android.

5 Autenticación y Seguridad

5.1 Mecanismo de Autenticación

El sistema útiliza autenticación basada en JSON Web Tokens (JWT) gestiónados a traves de Supabase Auth.

Flujo de autenticación

  1. El usuario introduce email y contraseña en la pantalla de login.
  2. El frontend envia POST /api/auth/login con las credenciales.
  3. El backend valida las credenciales contra Supabase Auth.
  4. Si son correctas, se devuelve un access token (JWT de corta duración) y un refresh token (de larga duración).
  5. El frontend almacena ambos tokens en sessionStorage: hola_token y hola_refresh.
  6. Todas las peticiones posteriores incluyen el access token en la cabecera Authorization: Bearer <token>.

5.2 Renovación Automática de Tokens

El sistema implementa renovación proactiva de tokens para sesiónes ininterrumpidas:

  • Al recibir un token, se calcula su fecha de expiración y se programa una renovación 2 minutos antes de que expire.
  • Si el token esta a menos de 1 minuto de expirar al cargar la página, se renueva inmediatamente.
  • Cuando el usuario vuelve a la pestaña (evento visibilitychange), se refresca el perfil.
  • Si la renovación falla, el usuario es desconectado automáticamente.

5.3 Cierre de Sesión

Al cerrar sesión se cancela el temporizador de renovación, se limpia sessionStorage y se redirige al login.

5.4 Protección de Rutas

Doble protección

Frontend: Componente ProtectedRoute que verifica autenticación y permisos de rol antes de renderizar.

Backend: Middleware authGuard (valida JWT) y adminGuard (restringe a admin/gestor).

5.5 Almacenamiento de Credenciales

  • Los tokens se almacenan en sessionStorage (se eliminan al cerrar la pestaña), no en localStorage, para mayor seguridad.
  • Las contraseñas nunca se almacenan en el frontend; son gestiónadas exclusivamente por Supabase Auth con hashing seguro.

6.2 Descripción Detallada de Cada Rol

Administrador (admin)

Acceso completo a todas las funcionalidades. Puede crear, editar y eliminar usuarios. Accede al panel de estadísticas. Gestióna empresas, dispositivos y tickets sin restricción. Administra canales de chat.

Gestor (gestor)

Mismos permisos que el administrador (acceso total a dashboard, tickets, usuarios, estadísticas, chat). Las diferencias clave son:

  • Recibe notificaciones por email cuando un cliente crea una nueva incidencia.
  • Es el único rol que puede cambiar el estado de un ticket a “Pendiente de facturar”.

Trabajador (trabajador)

Acceso al dashboard para consultar empresas y dispositivos. Gestióna tickets asignados: cambia estados, añade comentarios, registra horas. Usa el chat interno. No accede a estadísticas ni gestión de usuarios.

Cliente (cliente)

Acceso exclusivo al portal de incidencias (/incidencias). Puede crear nuevas incidencias y ver su estado. Los estados internos “Pendiente de facturar” y “Facturado” se muestran simplificados como “Completado”. No tiene acceso al chat ni a ninguna otra sección interna. Esta asociado obligatoriamente a una empresa. Si su empresa es matriz, ve tickets de todas las filiales.

8 Página de Inicio de Sesión

8.1 Descripción

La página de login (/login) es el punto de entrada al sistema. Presenta un formulario centrado con diseño limpio.

8.2 Elementos de la Interfaz

  • Campo de email: Input con validación de formato de correo electrónico.
  • Campo de contraseña: Input con botón de visibilidad (ícono de ojo) para mostrar/ocultar.
  • Botón “Acceder”: Botón principal que inicia el proceso de autenticación.
  • Mensajes de error: Se muestran debajo del formulario en caso de credenciales incorrectas.

8.3 Comportamiento

  1. El usuario introduce su email y contraseña.
  2. Al pulsar “Acceder”, se ejecuta login(email, password) del AuthContext.
  3. Si la autenticación es exitosa:
    • Clientes → /incidencias
    • Resto de roles → / (Dashboard)
  4. Si falla, se muestra un mensaje de error descriptivo.

Características principales

  • Colores personalizables por evento
  • Asignación múltiple de operarios con avatares
  • Vinculación a empresa, tickets y dispositivos
  • Avisos automáticos por email con antelación configurable

  • Contador de asignados en tiempo real

14.6 Vinculación de Empresa, Tickets y Dispositivos

  2. Automáticamente se cargan los tickets activos de esa empresa (estados distintos de “Cerrado” y “Pendiente de facturar”)
  3. También se cargan los dispositivos registrados para esa empresa
  4. Marca los tickets y dispositivos relevantes para ese evento

En la vista de detalle del evento se muestran los tickets vinculados con su número, asunto y estado, y los dispositivos con su nombre, categoría e IP.

19 Flujos de Trabajo Paso a Paso

Esta sección describe, paso a paso, cómo realizar las operaciones más comunes del sistema. Está pensada para que cualquier usuario nuevo pueda seguir los pasos sin conocimiento previo.

19.1 Dar de alta una empresa nueva

  1. Acceder al Dashboard (página principal, ruta /).
  2. Pulsar el botón “Nueva Empresa” (ícono +).
  3. Se abre un modal con tres pestañas:
    • Datos: Rellenar nombre (obligatorio), CIF (obligatorio), email, teléfono, dirección, estado y notas. Si esta empresa depende de otra, seleccionar la empresa matriz en el desplegable con búsqueda.
    • Servicios: Marcar los servicios contratados (Cloud, Soporte, Hardware, Redes, Seguridad, Backup).
    • Contactos: Pulsar “Añadir Contacto” y rellenar nombre, teléfono, email y cargo. Se pueden añadir múltiples contactos.
  4. Pulsar “Guardar”. La empresa aparecerá en el listado.

19.2 Registrar un dispositivo en una empresa

  1. En el Dashboard, pulsar sobre la tarjeta de la empresa deseada.
  2. Se abre la vista de dispositivos con pestañas por categoría (Equipos, Servidores, NAS, Redes, Web, Correos, Otros).
  3. Seleccionar la pestaña de la categoría correspondiente.
  4. Pulsar el botón “Añadir” dentro de la pestaña.
  5. Se abre un modal con los campos específicos de esa categoría. Ejemplo para un equipo:
    • Nombre del dispositivo (obligatorio) y tipo (PC, Portatil, etc.).
    • número de serie (obligatorio para equipos).
    • IP, AnyDesk ID, usuario y contraseña.
  6. Opcionalmente, añadir campos personalizados pulsando “Añadir” en la sección de campos extra (por ejemplo: “Ubicación” = “Oficina 3”).
  7. Pulsar “Guardar”.
Campos especiales para la categoría “Web”

Además de la URL, se pueden registrar: puerto, usuario FTP, contraseña FTP y contraseña del panel de administración. Estos se almacenan en campos_extra (JSONB).

19.3 Crear un ticket de soporte (como admin/gestor/trabajador)

  1. Ir a la página de Tickets (/tickets).
  2. Pulsar “Nuevo Ticket”.
  3. Se abre un modal con los siguientes campos:
    • Empresa (obligatorio): Selector con búsqueda. Al seleccionar una empresa, se cargan automáticamente sus contactos y dispositivos.
    • Contacto (opcional): Seleccionar de la lista de contactos de la empresa. también se puede crear un contacto nuevo pulsando “Crear contacto” (se abre un panel lateral sin cerrar el modal) o editar uno existente pulsando “Editar contacto”.
    • Dispositivos (opcional, selección múltiple): Lista de checkboxes con buscador. Se pueden seleccionar varios dispositivos afectados. también se puede crear un dispositivo nuevo pulsando “Añadir dispositivo” (panel lateral con selector de categoría y formulario).
    • Asunto (obligatorio): Título breve del problema.
    • Descripción (opcional): Detalles adicionales.
    • Prioridad: Baja / Media (por defecto) / Alta / Urgente.
    • Estado: Pendiente (por defecto) / En curso / Pausado / Completado / Pendiente de facturar (solo gestor) / Facturado.
    • Operarios: Selector múltiple con filtros rápidos (“Todos”, “Solo admins”, “Solo trabajadores”) y búsqueda.
  4. Pulsar “Guardar”. Los operarios asignados recibiran un email de notificación.

19.4 Flujo completo de un ticket: de creación a facturación

  1. Creación: Se crea el ticket con estado “Pendiente” y prioridad adecuada. Se asignan uno o varios operarios.
  2. Inicio del trabajo: El operario cambia el estado a “En curso”. El cronómetro de tiempo se activa automáticamente.
  3. Durante el trabajo: El operario puede:
    • Añadir comentarios con archivos adjuntos describiendo el progreso.
    • Subir archivos directamente al ticket (capturas, logs, etc.).
    • Registrar horas de trabajo con descripción y rango de fechas.
    • Escribir notas internas que solo ve el equipo (no el cliente).
    • Pausar el ticket si necesita esperar (estado “Pausado”): el cronómetro se pausa.
    • Reanudar el trabajo volviendo a “En curso”.
  4. Finalización: El operario cambia el estado a “Completado”. El cronómetro se detiene. El cliente recibe un email informando de la resolución.
  5. Facturación: El gestor cambia el estado a “Pendiente de facturar” (solo gestores pueden hacer esto). Tras emitir la factura, se cambia a “Facturado”.
Importante: Solo el gestor puede facturar

El estado “Pendiente de facturar” solo puede ser seleccionado por usuarios con rol gestor. Si otro rol tiene un ticket en ese estado, vera la opción deshabilitada en el formulario.

19.5 Crear una incidencia (como cliente)

  1. El cliente inicia sesión. Es redirigido automáticamente a /incidencias.
  2. Ve un listado con sus incidencias anteriores (si las tiene) o un mensaje “Todo en orden”.
  3. Pulsa “Nueva incidencia”.
  4. Se abre un modal con:
    • Sistema afectado (opcional): Grid de botones con los sistemas de su empresa (Equipos, Servidores, NAS, etc.). Puede elegir “Sin específicar”.
    • Asunto (obligatorio, máximo 200 carácteres): Descripción breve.
    • Descripción (opcional, máximo 2000 carácteres): Detalles del problema.
    • Archivos (opcional): Subida de imagenes, PDFs, documentos. Se muestra una lista con nombre y tamaño de cada archivo, con botón para eliminar antes de enviar.
  5. Pulsa “Enviar incidencia”. Aparece un mensaje de confirmación.
  6. Todos los gestores reciben un email notificandoles de la nueva incidencia.
  7. La incidencia aparece en el sistema de tickets para el equipo interno.

19.6 Qué ve el cliente tras crear la incidencia

  • La incidencia aparece en su listado con estado “Pendiente”.
  • Cuando el equipo la atiende, el estado cambia a “En curso”.
  • Cuando se completa, el cliente ve “Completado” (los estados “Pendiente de facturar” y “Facturado” se muestran simplificados como “Completado”).
  • Al pulsar sobre una incidencia, se abre un modal de detalle con: asunto, estado, fecha, descripción y archivos adjuntos con posibilidad de descarga.
  • Si la empresa del cliente es matriz y tiene filiales, el cliente vera las incidencias de todas las filiales además de las propias. En ese caso se muestra una columna extra “Empresa” en la tabla.

19.7 Crear un usuario nuevo (como admin)

  1. Ir a Usuarios (/usuarios).
  2. Pulsar “Nuevo Usuario”.
  3. Rellenar el formulario:
    • Nombre completo (obligatorio).
    • Email (obligatorio, debe ser único).
    • Rol:
      • Trabajador — acceso estandar.
      • Gestor — recibe incidencias de clientes, puede facturar.
      • Administrador — acceso total.
      • Cliente — portal de incidencias.
    • Empresa (obligatorio solo si el rol es “cliente”): Selector con búsqueda.
    • Teléfono (opcional, solo para clientes).
    • Contraseña (obligatoria en creación).
  4. Pulsar “Guardar”. El usuario ya puede iniciar sesión.

19.8 Usar el chat interno

  1. Ir a Chat (/chat).
  2. En el panel lateral izquierdo se ven los canales grupales y los mensajes directos.
  3. Para crear un canal: Pulsar el ícono “+” junto a “Canales”, poner nombre, descripción y seleccionar miembros.
  4. Para enviar un mensaje directo: Pulsar el ícono “+” junto a “Directos”, seleccionar la persona. Se envia una invitación que el destinatario debe aceptar.
  5. En la ventana de chat:
    • Escribir el mensaje en el campo inferior y pulsar Enter o el botón de envio.
    • Para adjuntar un archivo: Pulsar el ícono del clip.
    • Para mencionar a alguien: Escribir @ seguido del nombre. Aparece un selector de personas.
    • Para vincular un ticket: Usar el botón de ticket para buscar y seleccionar un ticket existente.
    • Clic derecho sobre un mensaje para: editar (solo propios), eliminar, anclar/desanclar.
    • Clic derecho sobre un canal para: fijar/desfijar, silenciar/activar notificaciones, ocultar/mostrar, editar (admin), eliminar.
  6. Los canales se pueden reordenar arrastrando y soltando (drag & drop).

19.9 Consultar estadísticas

  1. Ir a Estadísticas (/estadisticas) — solo admin y gestor.
  2. En la parte superior se ven 7 tarjetas KPI:
    • Total tickets — número global.
    • Abiertos — pendientes + en curso.
    • Sin asignar — sin operario asignado.
    • Completados — finalizados.
    • Facturados — cerrados y facturados.
    • Urgentes — con prioridad urgente.
    • últimos 7 días — creados en la ultima semana.
  3. Debajo, tres pestañas: Tickets, Operarios, Empresas.
  4. Usar el filtro de fechas (Desde/Hasta) en la cabecera para acotar el rango de datos. Pulsar “Todo” para quitar los filtros.

20 Funcionalidades Avanzadas

20.1 Creación de contactos y dispositivos desde el modal de ticket

Al crear o editar un ticket, no es necesario salir del modal para crear contactos o dispositivos nuevos:

  • Crear contacto: Pulsar “Crear contacto” junto al campo de contacto. Se despliega un panel lateral a la derecha del modal con el formulario (nombre, teléfono, email, cargo). El contacto se añade a la empresa selecciónada y queda disponible inmediatamente.
  • Editar contacto: Si hay un contacto seleccionado, aparece el botón “Editar contacto” que abre otro panel lateral con los datos precargados.
  • Crear dispositivo: Pulsar “Añadir dispositivo” junto a la lista de dispositivos. Se abre un panel lateral con dos pasos: primero seleccionar la categoría (grid de botones) y luego rellenar el formulario específico de esa categoría.
Panel lateral “side panel”

Estos paneles laterales aparecen junto al formulario principal sin cerrarlo, permitiendo completar la operación auxiliar y continuar con la creación del ticket de forma fluida.

20.2 Selección múltiple de dispositivos en tickets

Un ticket puede estar asociado a varios dispositivos simultaneamente (no solo uno). Al crear/editar un ticket:

  • Se muestra una lista de checkboxes con todos los dispositivos de la empresa.
  • Un buscador permite filtrar por nombre o tipo.
  • Se puede seleccionar/deseleccionar haciendo clic en cada dispositivo.
  • Un contador muestra cuántos estan seleccionados, con botón “Limpiar”.

20.3 Campos personalizados en dispositivos

Todos los tipos de dispositivos soportan campos personalizados (campos_extra). Son pares clave-valor libres que permiten almacenar información adicional no prevista en el formulario estandar. Ejemplos:

  • “Ubicación” = “Planta 2, Rack B”
  • “Garantia hasta” = “2027-03-15”
  • “Proveedor” = “Dell Technologies”

20.4 Jerarquía de empresas (matriz y filiales)

El sistema soporta relaciones matriz-filial entre empresas:

  • Al crear/editar una empresa, se puede seleccionar una empresa matriz mediante un dropdown con búsqueda.
  • Una empresa que es filial no puede seleccionarse como matriz de otra (se excluye de la lista).
  • Impacto en clientes: Un cliente vinculado a una empresa matriz ve los tickets de todas sus filiales. Un cliente vinculado a una filial solo ve los de su propia empresa.
  • En el Dashboard, las empresas filiales se visualizan asociadas a su matriz.

20.5 Sistema de contactos de empresas

Cada empresa puede tener múltiples contactos almacenados en formato JSON. Cada contacto tiene: nombre, teléfono, email y cargo. Los contactos son accesibles desde:

  • El detalle de empresa en el Dashboard.
  • El formulario de tickets, donde se puede seleccionar un contacto como referencia para ese ticket. Los datos del contacto (nombre, teléfono) se copian al ticket.

23 Preguntas Frecuentes (FAQ)

Generales

¿Qué necesito para acceder al sistema?

Un navegador web moderno (Chrome, Edge, Firefox o Safari) y unas credenciales (email y contraseña) proporcionadas por un administrador. No se requiere instalar ningun software adicional.

Se me ha cerrado la sesión sin hacer nada, por que?

Los tokens de sesión se almacenan en sessionStorage. Esto significa que si cierras la pestaña o el navegador, la sesión se pierde y tendrás que volver a iniciar sesión. Dentro de una misma pestaña, la sesión se renueva automáticamente mientras no cierres el navegador.

He olvidado mi contraseña, como la recupero?

Contacta con un administrador del sistema para que restablezca tu contraseña desde la página de gestión de usuarios.

Empresas y dispositivos

¿Qué es una empresa matriz y una filial?

Una empresa matriz es la empresa principal de un grupo. Las filiales son empresas que dependen de ella. En el sistema, al crear una empresa puedes asignarle una empresa matriz. Esto afecta a la visibilidad: un cliente de una empresa matriz puede ver los tickets de todas sus filiales.

Puedo añadir campos que no aparecen en el formulario del dispositivo?

Sí. Todos los formularios de dispositivos tienen una sección “Campos personalizados” donde puedes añadir pares campo/valor libres. Por ejemplo: “Ubicación” = “Rack A3”.

¿Qué tipos de dispositivos puedo registrar?

Equipos (PCs, portatiles, impresoras...), servidores (físicos, virtuales), NAS (Synology, QNAP), dispositivos de red (routers, switches, APs, firewalls), cuentas de correo, páginas web y otros no categorizados.

Tickets

Puedo asignar varios operarios a un mismo ticket?

Sí. Al crear o editar un ticket, el selector de operarios permite selección múltiple. Cada operario asignado recibe un email de notificación.

Puedo asociar un ticket a varios dispositivos?

Sí. El selector de dispositivos es de selección múltiple (checkboxes). Puedes buscar y seleccionar tantos como necesites.

¿Qué es una nota interna?

Las notas internas son textos que solo puede ver el equipo interno (admin, gestor, trabajador). Los clientes nunca ven las notas internas. Son útiles para anotar información sensible o instrucciones entre compañeros.

¿Por qué no puedo cambiar un ticket a “Pendiente de facturar”?

Solo los usuarios con rol gestor pueden cambiar un ticket a ese estado. Si eres admin o trabajador, verás la opción deshabilitada.

¿Cómo funciona el cronómetro de tiempo?

Se activa automáticamente al pasar el ticket a “En curso” y se pausa al ponerlo en “Pausado” o “Pendiente”. El tiempo acumulado se muestra en la lista de tickets y en el detalle. No es necesario iniciarlo manualmente.

Puedo crear un contacto o dispositivo sin salir del formulario de ticket?

Sí. Junto a los campos de contacto y dispositivo hay botones “Crear contacto” y “Añadir dispositivo” que abren un panel lateral sin cerrar el formulario del ticket. Tras guardarlo, el nuevo elemento queda disponible inmediatamente.

Portal de clientes

¿Qué estados ve el cliente?

El cliente ve: Pendiente, En curso, Pausado y Completado. Los estados internos “Pendiente de facturar” y “Facturado” se muestran simplificados como “Completado”, ya que la información de facturación no es relevante para el cliente.

El cliente puede acceder al chat?

No. El chat es una herramienta exclusiva del equipo interno (admin, gestor, trabajador). Los clientes solo tienen acceso al portal de incidencias.

El cliente puede ver los comentarios internos de sus tickets?

El cliente no tiene acceso a la vista de detalle completa del ticket. Solo ve el asunto, descripción, estado (simplificado), fecha y archivos adjuntos que el mismo subio al crear la incidencia.

Chat

¿Cómo menciono a alguien en el chat?

Escribe @ seguido del nombre de la persona. Aparecerá un selector automático donde puedes elegir al usuario. Al mencionarlo, recibira una notificación destacada.

Puedo vincular un ticket en un mensaje de chat?

Sí. Usa el botón de ticket (ícono de ticket) en la barra de redacción. Aparecerá un buscador donde puedes buscar por número o asunto. El mensaje mostrará una tarjeta con la información del ticket.

¿Cómo silencio un canal que tiene mucha actividad?

Haz clic derecho sobre el canal en el panel lateral y selecciona “Silenciar notificaciones”. Seguiras viendo los mensajes, pero no recibirás notificaciones.

¿Cómo funcionan las notificaciones del navegador?

La primera vez que accedes al chat, el navegador te pedirá permiso para mostrar notificaciones. Si aceptas, recibirás notificaciones nativas incluso cuando la pestaña este en segundo plano. Si las rechazas, solo verás los contadores de no leidos dentro de la app.

Estadísticas

¿Qué tarjetas KPI se muestran en la cabecera?

Total de tickets, abiertos (pendientes + en curso), sin asignar, completados, facturados, urgentes, y creados en los últimos 7 días.

El filtro de fechas afecta a todas las graficas?

El filtro de fechas afecta a las pestañas de Operarios y Empresas. Las tarjetas KPI de la cabecera y la pestaña de Tickets muestran siempre el resumen global.
